🔍 Ollamaの確認

まず、Windowsの PowerShell（青い画面） を開き、Ollamaが動くかバージョン確認をします。

※まだ入れていない人は、先に Ollama公式サイト からWindows版をインストールしておいてください。

Windows PowerShell
ollama --version

正常なら ollama version is 0.1.xx と表示されます。確認できたら次へ進みます。

🛠️ 動作モードの選択（どちらか選んでクリック）

PCのスペックや目的に合わせて、どちらかのボタンを押してください：

✨ 軽量版 (2b) の設定手順
⚠️ 注意（自己責任ルート）：
この軽量版は、低スペックPCでも動くように、本来の仕様（7b）とは異なる軽いモデル（2b）をダウンロードし、プログラム側の指定名である gemma4:e2b という名前に無理やり名前を偽装して認識させているカスタムルートです。
そのため、診断の精度が大幅に落ちたり、時々おかしな日本語を出力することがあります。動作の軽さを最優先した「自己責任モード」として使用してください。

PowerShell（青い画面） で実行してください：

Windows PowerShell
ollama pull gemma:2b
"FROM gemma:2b" | Set-Content Modelfile
ollama create gemma4:e2b -f ./Modelfile

success と出ればOKです。

💎 本物 (7b) の設定手順
📢 グラフィックボードのスペックについて：
本物のGemma（7b）を動かすには、PCに高いスペック（VRAM/メインメモリ）が要求されます。もし動かしてみてPCが極端に重くなったり、エラーが出る場合は、左の「軽量版ルート」へ切り替えてください。

PowerShell（青い画面） で実行してください：

Windows PowerShell
ollama pull gemma4:e2b

設定が終わったら、初期設定はすべて完了です！2回目以降は、上のタブの「毎回の起動手順」だけをやればOKになります。

💡 【超重要】 学校から家に帰ったときなど、ネットワーク（Wi-Fi）の場所が変わったときも、必ずこの手順を上から順番にやり直してください！
1️⃣ Windows側：Ollamaのホスト設定＆起動

外部（WSL2）からの接続を許可するため、Windowsの PowerShell（青い画面） で以下の2行を必ず実行してください。

Windows PowerShell
$env:OLLAMA_HOST="0.0.0.0"
ollama serve

※ Listening on [::]:11434 と出れば待機完了です。この画面は閉じずにそのままにしておきます。

2️⃣ Ubuntu側（画面1）：自動IP追従中継マシンの起動

Ubuntu（黒い画面） を開き、以下のコマンドを実行します。Wi-Fiが変わったり再起動してWindowsのIPアドレスが勝手に変動しても、自動的に本物のIPを割り出して安全にリレーを回し続けます。

Ubuntu Terminal (中継専用画面)
sudo killall -9 socat 2>/dev/null
while true; do socat TCP-LISTEN:11434,reuseaddr TCP:$(host -t A $(hostname).local | awk '/has address/ {print $4; exit}'):11434; done

⚠️ 注意：これを実行すると画面が静かに待機状態になります（次のコマンドが打てなくなります）。これで大正解です！この画面はこのまま放置してください。

3️⃣ Ubuntu側（画面2）：アプリの起動

タスクバーから「もう1つ新しくUbuntu（黒い画面）」を開きます。ソースコードのあるフォルダ（~/skin/aiface）に移動し、仮想環境に入って起動します。

Ubuntu Terminal (アプリ起動画面)
cd ~/skin/aiface
source .venv/bin/activate
python3 -m streamlit run skinchecker/app/streamlit_app.py
💡 仮想環境チェック
2行目を打ったあと、画面の左端に (.venv) という文字が出ていることを必ず確認してください！

表示されたLocal URL（http://localhost:8501）をブラウザで開けば、家でも学校でも100%確実に肌診断が動き出します！

📁 【自分用メモ】別プロジェクト（顔UIビューアー）の起動手順

フォルダ aiface260508-wsl2 側で、画像を直接読み込んでUIビューアーを動かす時のコマンド一式です。

Ubuntu Terminal (aiface260508-wsl2用)
# 1. フォルダ移動と仮想環境アクティベート
cd ~/skin/aiface260508-wsl2
source .venv/bin/activate

# 2. PYTHONPATHを通してビューアーを実行（サンプル画像を読み込み）
PYTHONPATH=src python3 src/aiface/ui/viewer.py -i samples/face.jpg
※ Streamlitの肌診断とはインポート経路やフォルダが異なるので、作業を戻す時はこのコマンドをそのまま使用すること。
🔍 ファイル・コード内テキストの検索

「あのソースコードどこに置いたっけ？」「特定の関数を使っているファイルを探したい」という時に使えるコマンドです。

ファイル名で探す
Ubuntu
find . -name "app.py"
現在地（.）以下から、ファイル名が完全に一致するものを検索。
特定の拡張子を全検索
Ubuntu
find . -name "*.py"
現在地以下のすべてのPythonファイルをリストアップ。
💡 激重な .venv を無視して検索
Ubuntu
find . -path "*/.venv*" -prune -o -name "*.py" -print
仮想環境の中の膨大な外部ライブラリを検索対象から除外し、自分が書いたファイルだけを見つけます！
コード内の文字を全検索
Ubuntu
grep -rn "OLLAMA_HOST" .
現在地以下の全ファイルを対象に、指定した文字列を検索して行番号と一緒に表示。
.venv を無視して文字検索
Ubuntu
grep -rn --exclude-dir=.venv "import streamlit" .
ライブラリのコードを無視して、自分のソースコードだけから文字を探せます。
🧹 プロセスのお掃除（不具合リセット）

「なんか通信が詰まった」「中継をやり直したい」という時に、裏で暴走したプロセスを一括で黙らせるお掃除用コマンドです。

Ubuntu Terminal
# ゾンビ化した中継プロセスを全滅させる
sudo killall -9 socat 2>/dev/null

# 現在の画面の裏(&)で動いているジョブをすべて強制終了する
kill -9 $(jobs -p) 2>/dev/null

📩 お困りの際は（個別サポート・質問箱）

手順通りに進めても (.venv) が表示されない場合や、画面にエラーが出る際などにこちらをご活用ください。配慮の至らない点や書き損ねている部分も多々あるかと存じますので、お気づきの点がございましたらご指摘いただけますと幸いです。

不具合の原因を特定するため、フォームからは画面の赤文字だけでなく、UbuntuやPowerShellのログ（ターミナルのテキスト）も一緒にコピーして送っていただけますと幸いです。返答が滞っている（忘れていそうな）場合は、大変恐縮ですが口頭で直接お声がけいただけますと助かります。
